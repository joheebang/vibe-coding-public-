# PM Tool — 게임 업데이트 콘텐츠 관리 시스템

게임 서비스의 업데이트 마일스톤을 팀 전체가 한 화면에서 협업 관리할 수 있는 PM 도구입니다.

## 주요 기능

- **역할 기반 편집** — PM / UI / 시스템 / 밸런스 / 클라 / 서버 / QA 역할별로 편집 가능한 열이 구분됩니다.
- **실시간 공유** — 같은 네트워크 내 팀원 누구나 브라우저로 접속해 입력하고 자동 저장됩니다.
- **Jira 연동** — 티켓 상태 자동 조회, 상위/하위 티켓 일괄 생성, Fix Version 관리
- **Slack 연동** — 슬랙 쓰레드를 퍼머링크로 불러와 상세 내용에 붙여넣기
- **엑셀 Import/Export** — xlsx 파일로 데이터를 올리거나 내려받을 수 있습니다.
- **백업 관리** — 최신 30개 자동 보관, 언제든 복원 가능
- **다크 모드** — 라이트/다크 테마 전환

## 기술 스택

- **Backend**: Node.js + Express
- **Frontend**: Vanilla JS + HTML/CSS (no framework)
- **Data Storage**: JSON 파일
- **Integrations**: Jira REST API v3, Slack API

## 시작하기

```bash
# 1. 의존성 설치
npm install

# 2. 환경변수 설정
cp .env.example .env
# .env 파일을 열어 Jira / Slack 정보 입력

# 3. 서버 실행
npm start
# → http://localhost:3001
```

## 환경변수

| 변수 | 설명 | 필수 |
|---|---|---|
| `PORT` | 서버 포트 (기본: 3001) | |
| `PM_PASSWORD` | PM 역할 진입 비밀번호 | |
| `JIRA_DOMAIN` | Jira 도메인 (예: `your-org.atlassian.net`) | Jira 사용 시 |
| `JIRA_EMAIL` | Jira 계정 이메일 | Jira 사용 시 |
| `JIRA_API_TOKEN` | Jira API 토큰 | Jira 사용 시 |
| `JIRA_PROJECT` | Jira 프로젝트 키 (예: `MYPROJ`) | Jira 사용 시 |
| `SLACK_BOT_TOKEN` | Slack Bot Token | Slack 사용 시 |

## 화면 구성

```
┌──────────────┬────────────────────────────────────────┐
│   사이드바    │           메인 콘텐츠                   │
│              │                                        │
│ 📅 목표일자   │  [ M1 ] 작업 목표일자 확인              │
│ 📊 진행상황   │  ┌──────────────────────────────────┐  │
│ 🎫 Jira생성  │  │  일정표 / 콘텐츠 테이블 (역할별 열) │  │
│ 💾 백업관리   │  └──────────────────────────────────┘  │
└──────────────┴────────────────────────────────────────┘
```

## 사용 흐름

1. PM이 서버를 실행하고 팀원들에게 접속 주소 공유
2. 팀원 각자 브라우저에서 접속 → 역할 선택
3. 본인 열에 작업 완료 예상 빌드 번호 입력 → 자동 저장
4. PM은 Jira 티켓 일괄 생성, 일정 관리

## 프로젝트 배경

마일스톤마다 각 파트에 개별 문의 → 취합 → Jira 생성의 반복을 줄이기 위해 제작했습니다.
팀원 전체가 같은 화면을 보며 입력하는 방식으로 취합 시간을 단축했습니다.
